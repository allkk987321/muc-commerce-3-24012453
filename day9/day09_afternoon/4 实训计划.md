## **二、实训推进原则**
### **（一）项目驱动原则**
每天的实训任务都围绕“校园轻集市”的具体项目功能展开，而不是孤立讲解技术点。学生应在完成页面、接口、状态、交互和图表等具体任务的过程中学习和巩固相关技术。

### **（二）功能深度控制原则**
本项目是教学型前端实训项目，不追求真实产品级复杂度。二手交易、失物招领、拼单搭子、跑腿委托四类场景应完整覆盖，但每类场景均控制在前端可实现、可演示、可验收的范围内。

其中，模拟砍价、消息回复、信用分、趋势统计、安全提醒等功能均属于教学模拟，不涉及真实支付、订单、权限、审核、WebSocket 和复杂后端业务逻辑。

### **（三）AI辅助开发原则**
AI Coding 工具可以用于需求拆解、页面生成、组件编写、接口封装、错误排查、代码优化和测试用例设计。但学生不能直接复制 AI 输出结果而不理解，应对 AI 生成代码进行阅读、修改、运行和验证。

### **（四）Git版本保护原则**
每天至少完成一次稳定版本提交。在使用 AI 对项目进行较大范围修改之前，必须先提交当前可运行版本。若 AI 修改导致代码混乱或功能失效，应能够通过 Git 查看提交历史并回退到稳定版本。

### **（五）每日验收原则**
每天结束前必须完成阶段性验收。验收内容包括：当天功能是否完成、项目是否能运行、关键页面是否能访问、接口是否能返回数据、Git 是否完成提交、问题是否记录清楚。

<font style="color:#DF2A3F;">注：本章中的验收标准为阶段性实训验收标准，用于判断学生当天任务是否完成；各功能模块的最终实现要求以《2 需求分析》中的核心功能验收标准为准。</font>

## **十四、实训计划总结**
本实训计划以 7 天为周期，将“校园轻集市”项目从需求理解、页面开发、Mock API、接口通信、状态管理、图表展示、测试优化到答辩交付进行完整安排。与传统按技术点推进的课程不同，本计划强调每天都围绕项目交付成果展开，使学生能够在真实业务场景中理解技术的作用。

通过本实训，学生不仅能够完成一个可运行的 Vue3 前端项目，还能够理解前端工程化开发的基本流程，掌握前端与 Mock API 的数据交互方式，体验 AI Coding 工具在开发中的效率提升作用，并通过 Git 形成基本版本管理和风险控制意识。

“校园轻集市”项目的最终目标不是做一个真实上线的校园交易平台，而是帮助学生在 AI 时代的软件开发环境中，完成一次完整、可控、可展示的前端工程实践。
